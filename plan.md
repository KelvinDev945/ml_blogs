# 实施计划

1. TODO 梳理与模板分析  
   - 完成 `TODO.md` 中的优先事项，明确阶段目标与约束。  
   - 深入分析模板仓库配置、构建流程与样式体系。

2. GitHub Pages 搜索首页  
   - 搭建 `docs/index.html` 搜索首页，复用模板组件并调整为项目需求。  
   - 本地验证搜索交互与样式，并确保可顺利部署到 GitHub Pages。

3. 内容抓取与初筛  
   - 编写脚本抓取 `https://spaces.ac.cn/feed`，按标题判定数学/机器学习相关性。  
   - 将相关文章保存到 `blogs_raw/<slug>.md`，并在 `blog_list.txt` 记录路径与“已完成/未完成”状态。

4. 公式推导与注释增强  
   - 在 `blogs_raw` 中补充完整的公式推导与注释，标注完成情况。  
   - 对尚未完善的条目保留“未完成”以待后续迭代。

5. 文章页面生成与迭代  
   - 基于模板生成 `docs/posts/<slug>.html`，逐篇上线文章。  
   - 每次更新同步 `blog_list.txt` 状态并检查导航与样式。

6. 站点整体验证  
   - 全面检查首页、文章页、搜索与链接，准备 GitHub Pages 发布。
