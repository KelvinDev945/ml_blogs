生成扩散模型漫谈（一）：DDPM = 拆楼 + 建楼¶

说到生成模型，VAE、GAN可谓是“如雷贯耳”，本站也有过多次分享。此外，还有一些比较小众的选择，如flow模型、VQ-VAE等，也颇有人气，尤其是VQ-VAE及其变体VQ-GAN，近期已经逐渐发展到“图像的Tokenizer”的地位，用来直接调用NLP的各种预训练方法。除了这些之外，还有一个本来更小众的选择——扩散模型（Diffusion Models）——正在生成模型领域“异军突起”，当前最先进的两个文本生成图像——OpenAI的DALL·E 2和Google的Imagen，都是基于扩散模型来完成的。

从本文开始，我们开一个新坑，逐渐介绍一下近两年关于生成扩散模型的一些进展。据说生成扩散模型以数学复杂闻名，似乎比VAE、GAN要难理解得多，是否真的如此？扩散模型真的做不到一个“大白话”的理解？让我们拭目以待。

新的起点¶

其实我们在之前的文章《能量视角下的GAN模型（三）：生成模型=能量模型》、《从去噪自编码器到生成模型》也简单介绍过扩散模型。说到扩散模型，一般的文章都会提到能量模型（Energy-based Models）、得分匹配（Score Matching）、朗之万方程（Langevin Equation）等等，简单来说，是通过得分匹配等技术来训练能量模型，然后通过郎之万方程来执行从能量模型的采样。

从理论上来讲，这是一套很成熟的方案，原则上可以实现任何连续型对象（语音、图像等）的生成和采样。但从实践角度来看，能量函数的训练是一件很艰难的事情，尤其是数据维度比较大（比如高分辨率图像）时，很难训练出完备能量函数来；另一方面，通过朗之万方程从能量模型的采样也有很大的不确定性，得到的往往是带有噪声的采样结果。所以很长时间以来，这种传统路径的扩散模型只是在比较低分辨率的图像上做实验。

如今生成扩散模型的大火，则是始于2020年所提出的DDPM（Denoising Diffusion Probabilistic Model），虽然也用了“扩散模型”这个名字，但事实上除了采样过程的形式有一定的相似之外，DDPM与传统基于朗之万方程采样的扩散模型可以说完全不一样，这完全是一个新的起点、新的篇章。

准确来说，DDPM叫“渐变模型”更为准确一些，扩散模型这一名字反而容易造成理解上的误解，传统扩散模型的能量模型、得分匹配、朗之万方程等概念，其实跟DDPM及其后续变体都没什么关系。有意思的是，DDPM的数学框架其实在ICML2015的论文《Deep Unsupervised Learning using Nonequilibrium Thermodynamics》就已经完成了，但DDPM是首次将它在高分辨率图像生成上调试出来了，从而引导出了后面的火热。由此可见，一个模型的诞生和流行，往往还需要时间和机遇，

拆楼建楼¶

很多文章在介绍DDPM时，上来就引入转移分布，接着就是变分推断，一堆数学记号下来，先吓跑了一群人（当然，从这种介绍我们可以再次看出，DDPM实际上是VAE而不是扩散模型），再加之人们对传统扩散模型的固有印象，所以就形成了“需要很高深的数学知识”的错觉。事实上，DDPM也可以有一种很“大白话”的理解，它并不比有着“造假-鉴别”通俗类比的GAN更难。

首先，我们想要做一个像GAN那样的生成模型，它实际上是将一个随机噪声$\boldsymbol{z}$变换成一个数据样本$\boldsymbol{x}$的过程：

我们可以将这个过程想象为“建设”，其中随机噪声$\boldsymbol{z}$是砖瓦水泥等原材料，样本数据$\boldsymbol{x}$是高楼大厦，所以生成模型就是一支用原材料建设高楼大厦的施工队。

这个过程肯定很难的，所以才有了那么多关于生成模型的研究。但俗话说“破坏容易建设难”，建楼你不会，拆楼你总会了吧？我们考虑将高楼大厦一步步地拆为砖瓦水泥的过程：设$\boldsymbol{x}0$为建好的高楼大厦（数据样本），$\boldsymbol{x}_T$为拆好的砖瓦水泥（随机噪声），假设“拆楼”需要$T$步，整个过程可以表示为
\begin{equation}\boldsymbol{x} = \boldsymbol{x}_0 \to \boldsymbol{x}_1 \to \boldsymbol{x}_2 \to \cdots \to \boldsymbol{x}} \to \boldsymbol{xT = \boldsymbol{z}\end{equation}
建高楼大厦的难度在于，从原材料$\boldsymbol{x}_T$到最终高楼大厦$\boldsymbol{x}_0$的跨度过大，普通人很难理解$\boldsymbol{x}_T$是怎么一下子变成$\boldsymbol{x}_0$的。但是，当我们有了“拆楼”的中间过程$\boldsymbol{x}_1,\boldsymbol{x}_2,\cdots,\boldsymbol{x}_T$后，我们知道$\boldsymbol{x}} \to \boldsymbol{xt$代表着拆楼的一步，那么反过来$\boldsymbol{x}_t\to \boldsymbol{x}}$不就是建楼的一步？如果我们能学会两者之间的变换关系$\boldsymbol{x{t-1}=\boldsymbol{\mu}(\boldsymbol{x}_t)$，那么从$\boldsymbol{x}_T$出发，反复地执行$\boldsymbol{x}}=\boldsymbol{\mu}(\boldsymbol{xT)$、$\boldsymbol{x}_0$出来？}=\boldsymbol{\mu}(\boldsymbol{x}_{T-1})$、...，最终不就能造出高楼大厦$\boldsymbol{x

该怎么拆¶

正所谓“饭要一口一口地吃”，楼也要一步一步地建，DDPM做生成模型的过程，其实跟上述“拆楼-建楼”的类比是完全一致的，它也是先反过来构建一个从数据样本渐变到随机噪声的过程，然后再考虑其逆变换，通过反复执行逆变换来完成数据样本的生成，所以本文前面才说DDPM这种做法其实应该更准确地称为“渐变模型”而不是“扩散模型”。

具体来说，DDPM将“拆楼”的过程建模为
\begin{equation}\boldsymbol{x}t = \alpha_t \boldsymbol{x}} + \beta_t \boldsymbol{\varepsilont,\quad \boldsymbol{\varepsilon}_t\sim\mathcal{N}(\boldsymbol{0}, \boldsymbol{I})\label{eq:forward}\end{equation}
其中有$\alpha_t,\beta_t > 0$且$\alpha_t^2 + \beta_t^2=1$，$\beta_t$通常很接近于0，代表着单步“拆楼”中对原来楼体的破坏程度，噪声$\boldsymbol{\varepsilon}_t$的引入代表着对原始信号的一种破坏，我们也可以将它理解为“原材料”，即每一步“拆楼”中我们都将$\boldsymbol{x}_t$的原料”。（}$拆解为“$\alpha_t \boldsymbol{x}_{t-1}$的楼体 + $\beta_t \boldsymbol{\varepsilon提示： 本文$\alpha_t,\beta_t$的定义跟原论文不一样。）

反复执行这个拆楼的步骤，我们可以得到：
\begin{equation}\begin{aligned} \boldsymbol{x}t =&\, \alpha_t \boldsymbol{x}} + \beta_t \boldsymbol{\varepsilont \\ =&\, \alpha_t \big(\alpha} \boldsymbol{x{t-2} + \beta} \boldsymbol{\varepsilon{t-1}\big) + \beta_t \boldsymbol{\varepsilon}_t \\ =&\,\cdots\\ =&\,(\alpha_t\cdots\alpha_1) \boldsymbol{x}_0 + \underbrace{(\alpha_t\cdots\alpha_2)\beta_1 \boldsymbol{\varepsilon}_1 + (\alpha_t\cdots\alpha_3)\beta_2 \boldsymbol{\varepsilon}_2 + \cdots + \alpha_t\beta} \boldsymbol{\varepsilon{t-1} + \beta_t \boldsymbol{\varepsilon}_t}} \end{aligned}\label{eq:expand}\end{equation}
可能刚才读者就想问为什么叠加的系数要满足$\alpha_t^2 + \beta_t^2 = 1$了，现在我们就可以回答这个问题。首先，式中花括号所指出的部分，正好是多个独立的正态噪声之和，其均值为0，方差则分别为$(\alpha_t\cdots\alpha_2)^2\beta_1^2$、$(\alpha_t\cdots\alpha_3)^2\beta_2^2$、...、$\alpha_t^2\beta_{t-1}^2$、$\beta_t^2$；然后，我们利用一个概率论的知识——正态分布的叠加性，即上述多个独立的正态噪声之和的分布，实际上是均值为0、方差为$(\alpha_t\cdots\alpha_2)^2\beta_1^2 + (\alpha_t\cdots\alpha_3)^2\beta_2^2 + \cdots + \alpha_t^2\beta_{t-1}^2 + \beta_t^2$的正态分布；最后，在$\alpha_t^2 + \beta_t^2 = 1$恒成立之下，我们可以得到式$\eqref{eq:expand}$的各项系数平方和依旧为1，即
\begin{equation}(\alpha_t\cdots\alpha_1)^2 + (\alpha_t\cdots\alpha_2)^2\beta_1^2 + (\alpha_t\cdots\alpha_3)^2\beta_2^2 + \cdots + \alpha_t^2\beta_{t-1}^2 + \beta_t^2 = 1\end{equation}
所以实际上相当于有
\begin{equation}\boldsymbol{x}t = \underbrace{(\alpha_t\cdots\alpha_1)}}\bar{\alphat} \boldsymbol{x}_0 + \underbrace{\sqrt{1 - (\alpha_t\cdots\alpha_1)^2}}}\bar{\beta}_t} \bar{\boldsymbol{\varepsilon}}_t,\quad \bar{\boldsymbol{\varepsilon}}_t\sim\mathcal{N}(\boldsymbol{0}, \boldsymbol{I})\label{eq:skip}\end{equation 这就为计算$\boldsymbol{x}_t$提供了极大的便利。另一方面，DDPM会选择适当的$\alpha_t$形式，使得有$\bar{\alpha}_T\approx 0$，这意味着经过$T$步的拆楼后，所剩的楼体几乎可以忽略了，已经全部转化为原材料$\boldsymbol{\varepsilon}$。（提示： 本文$\bar{\alpha}_t$的定义跟原论文不一样。）

又如何建¶

“拆楼”是$\boldsymbol{x}{t-1}\to \boldsymbol{x}_t$的过程，这个过程我们得到很多的数据对$(\boldsymbol{x}},\boldsymbol{xt)$，那么“建楼”自然就是从这些数据对中学习一个$\boldsymbol{x}_t\to \boldsymbol{x}}$的模型。设该模型为$\boldsymbol{\mu}(\boldsymbol{xt)$，那么容易想到学习方案就是最小化两者的欧氏距离：
\begin{equation}\left\Vert\boldsymbol{x}} - \boldsymbol{\mu}(\boldsymbol{xt)\right\Vert^2\label{eq:loss-0}\end{equation}
其实这已经非常接近最终的DDPM模型了，接下来让我们将这个过程做得更精细一些。首先“拆楼”的式$\eqref{eq:forward}$可以改写为$\boldsymbol{x}} = \frac{1}{\alpha_t}\left(\boldsymbol{xt - \beta_t \boldsymbol{\varepsilon}_t\right)$，这启发我们或许可以将“建楼”模型$\boldsymbol{\mu}(\boldsymbol{x}_t)$设计成
\begin{equation}\boldsymbol{\mu}(\boldsymbol{x}_t) = \frac{1}{\alpha_t}\left(\boldsymbol{x}_t - \beta_t \boldsymbol{\epsilon}}}(\boldsymbol{xt, t)\right)\label{eq:sample}\end{equation}
的形式，其中$\boldsymbol{\theta}$是训练参数，将其代入到损失函数，得到
\begin{equation}\left\Vert\boldsymbol{x}} - \boldsymbol{\mu}(\boldsymbol{xt)\right\Vert^2 = \frac{\beta_t^2}{\alpha_t^2}\left\Vert \boldsymbol{\varepsilon}_t - \boldsymbol{\epsilon}}}(\boldsymbol{xt, t)\right\Vert^2\end{equation}
前面的因子$\frac{\beta_t^2}{\alpha_t^2}$代表loss的权重，这个我们可以暂时忽略，最后代入结合式$\eqref{eq:skip}$和$\eqref{eq:forward}$所给出$\boldsymbol{x}_t$的表达式
\begin{equation}\boldsymbol{x}_t = \alpha_t\boldsymbol{x}} + \beta_t \boldsymbol{\varepsilont = \alpha_t\left(\bar{\alpha}}\boldsymbol{x0 + \bar{\beta}}\bar{\boldsymbol{\varepsilon}{t-1}\right) + \beta_t \boldsymbol{\varepsilon}_t = \bar{\alpha}_t\boldsymbol{x}_0 + \alpha_t\bar{\beta}}\bar{\boldsymbol{\varepsilon}{t-1} + \beta_t \boldsymbol{\varepsilon}_t \end{equation}
得到损失函数的形式为
\begin{equation}\left\Vert \boldsymbol{\varepsilon}_t - \boldsymbol{\epsilon}}}(\bar{\alphat\boldsymbol{x}_0 + \alpha_t\bar{\beta}}\bar{\boldsymbol{\varepsilon}}_{t-1} + \beta_t \boldsymbol{\varepsilon}_t, t)\right\Vert^2\label{eq:loss-1}\end{equation 可能读者想问为什么要回退一步来给出$\boldsymbol{x}_t$，直接根据式$\eqref{eq:skip}$来给出$\boldsymbol{x}_t$可以吗？答案是不行，因为我们已经事先采样了$\boldsymbol{\varepsilon}_t$，而$\boldsymbol{\varepsilon}_t$跟$\bar{\boldsymbol{\varepsilon}}_t$不是相互独立的，所以给定$\boldsymbol{\varepsilon}_t$的情况下，我们不能完全独立地采样$\bar{\boldsymbol{\varepsilon}}_t$。

降低方差¶

原则上来说，损失函数$\eqref{eq:loss-1}$就可以完成DDPM的训练，但它在实践中可能有方差过大的风险，从而导致收敛过慢等问题。要理解这一点并不困难，只需要观察到式$\eqref{eq:loss-1}$实际上包含了4个需要采样的随机变量：

1、从所有训练样本中采样一个$\boldsymbol{x}_0$；

2、从正态分布$\mathcal{N}(\boldsymbol{0}, \boldsymbol{I})$中采样$\bar{\boldsymbol{\varepsilon}}_{t-1}, \boldsymbol{\varepsilon}_t$（两个不同的采样结果）；

3、从$1\sim T$中采样一个$t$。

要采样的随机变量越多，就越难对损失函数做准确的估计，反过来说就是每次对损失函数进行估计的波动（方差）过大了。很幸运的是，我们可以通过一个积分技巧来将$\bar{\boldsymbol{\varepsilon}}_{t-1}, \boldsymbol{\varepsilon}_t$合并成单个正态随机变量，从而缓解一下方差大的问题。

这个积分确实有点技巧性，但也不算复杂。由于正态分布的叠加性，我们知道$\alpha_t\bar{\beta}{t-1}\bar{\boldsymbol{\varepsilon}}} + \beta_t \boldsymbol{\varepsilont$实际上相当于单个随机变量$\bar{\beta}_t\boldsymbol{\varepsilon}|\boldsymbol{\varepsilon}\sim \mathcal{N}(\boldsymbol{0}, \boldsymbol{I})$，同理$\beta_t \bar{\boldsymbol{\varepsilon}}$，所以这是两个相互独立的正态随机变量。} - \alpha_t\bar{\beta}_{t-1} \boldsymbol{\varepsilon}_t$实际上相当于单个随机变量$\bar{\beta}_t\boldsymbol{\omega}|\boldsymbol{\omega}\sim \mathcal{N}(\boldsymbol{0}, \boldsymbol{I})$，并且可以验证$\mathbb{E}[\boldsymbol{\varepsilon}\boldsymbol{\omega}^{\top}]=\boldsymbol{0

接下来，我们反过来将$\boldsymbol{\varepsilon}t$用$\boldsymbol{\varepsilon},\boldsymbol{\omega}$重新表示出来
\begin{equation}\boldsymbol{\varepsilon}_t = \frac{(\beta_t \boldsymbol{\varepsilon} - \alpha_t\bar{\beta}} \boldsymbol{\omega})\bar{\betat}{\beta_t^2 + \alpha_t^2\bar{\beta}}^2} = \frac{\beta_t \boldsymbol{\varepsilon} - \alpha_t\bar{\beta{t-1} \boldsymbol{\omega}}{\bar{\beta}_t}\end{equation}
代入到式$\eqref{eq:loss-1}$得到
\begin{equation}\begin{aligned} &\,\mathbb{E}}{t-1}, \boldsymbol{\varepsilon}_t\sim \mathcal{N}(\boldsymbol{0}, \boldsymbol{I})}\left[\left\Vert \boldsymbol{\varepsilon}_t - \boldsymbol{\epsilon}}}(\bar{\alphat\boldsymbol{x}_0 + \alpha_t\bar{\beta}}\bar{\boldsymbol{\varepsilon}{t-1} + \beta_t \boldsymbol{\varepsilon}_t, t)\right\Vert^2\right] \\ =&\,\mathbb{E}}, \boldsymbol{\varepsilon}\sim \mathcal{N}(\boldsymbol{0}, \boldsymbol{I})}\left[\left\Vert \frac{\beta_t \boldsymbol{\varepsilon} - \alpha_t\bar{\beta{t-1} \boldsymbol{\omega}}{\bar{\beta}_t} - \boldsymbol{\epsilon}}}(\bar{\alphat\boldsymbol{x}_0 + \bar{\beta}_t\boldsymbol{\varepsilon}, t)\right\Vert^2\right] \end{aligned}\end{equation}
注意到，现在损失函数关于$\boldsymbol{\omega}$只是二次的，所以我们可以展开然后将它的期望直接算出来，结果是
\begin{equation}\frac{\beta_t^2}{\bar{\beta}_t^2}\mathbb{E}}\sim \mathcal{N}(\boldsymbol{0}, \boldsymbol{I})}\left[\left\Vert\boldsymbol{\varepsilon} - \frac{\bar{\betat}{\beta_t}\boldsymbol{\epsilon}}}(\bar{\alphat\boldsymbol{x}_0 + \bar{\beta}_t\boldsymbol{\varepsilon}, t)\right\Vert^2\right]+\text{常数}\end{equation}
再次省掉常数和损失函数的权重，我们得到DDPM最终所用的损失函数：
\begin{equation}\left\Vert\boldsymbol{\varepsilon} - \frac{\bar{\beta}_t}{\beta_t}\boldsymbol{\epsilon}}}(\bar{\alphat\boldsymbol{x}_0 + \bar{\beta}_t\boldsymbol{\varepsilon}, t)\right\Vert^2\end{equation}
（提示： 原论文中的$\boldsymbol{\epsilon}}}$实际上就是本文的$\frac{\bar{\betat}{\beta_t}\boldsymbol{\epsilon}$，所以大家的结果是完全一样的。）}

递归生成¶

至此，我们算是把DDPM的整个训练流程捋清楚了。内容写了不少，你要说它很容易，那肯定说不上，但真要说非常困难的地方也几乎没有——没有用到传统的能量函数、得分匹配等工具，甚至连变分推断的知识都没有用到，只是借助“拆楼-建楼”的类比和一些基本的概率论知识，就能得到完全一样的结果。所以说，以DDPM为代表的新兴起的生成扩散模型，实际上没有很多读者想象的复杂，它可以说是我们从“拆解-重组”的过程中学习新知识的形象建模。

训练完之后，我们就可以从一个随机噪声$\boldsymbol{x}T\sim\mathcal{N}(\boldsymbol{0}, \boldsymbol{I})$出发执行$T$步式$\eqref{eq:sample}$来进行生成：
\begin{equation}\boldsymbol{x}} = \frac{1}{\alpha_t}\left(\boldsymbol{xt - \beta_t \boldsymbol{\epsilon}}}(\boldsymbol{xt, t)\right)\end{equation}
这对应于自回归解码中的Greedy Search。如果要进行Random Sample，那么需要补上噪声项：
\begin{equation}\boldsymbol{x}} = \frac{1}{\alpha_t}\left(\boldsymbol{xt - \beta_t \boldsymbol{\epsilon}}}(\boldsymbol{x}_t, t)\right) + \sigma_t \boldsymbol{z},\quad \boldsymbol{z}\sim\mathcal{N}(\boldsymbol{0}, \boldsymbol{I})\end{equation 一般来说，我们可以让$\sigma_t=\beta_t$，即正向和反向的方差保持同步。这个采样过程跟传统扩散模型的朗之万采样不一样的地方在于：DDPM的采样每次都从一个随机噪声出发，需要重复迭代$T$步来得到一个样本输出；朗之万采样则是从任意一个点出发，反复迭代无限步，理论上这个迭代无限步的过程中，就把所有数据样本都被生成过了。所以两者除了形式相似外，实质上是两个截然不同的模型。

从这个生成过程中，我们也可以感觉到它其实跟Seq2Seq的解码过程是一样的，都是串联式的自回归生成，所以生成速度是一个瓶颈，DDPM设了$T=1000$，意味着每生成一个图片，需要将$\boldsymbol{\epsilon}_{\boldsymbol{\theta}}(\boldsymbol{x}_t, t)$反复执行1000次，因此DDPM的一大缺点就是采样速度慢，后面有很多工作都致力于提升DDPM的采样速度。而说到“图片生成 + 自回归模型 + 很慢”，有些读者可能会联想到早期的PixelRNN、PixelCNN等模型，它们将图片生成转换成语言模型任务，所以同样也是递归地进行采样生成以及同样地慢。那么DDPM的这种自回归生成，跟PixelRNN/PixelCNN的自回归生成，又有什么实质区别呢？为什么PixelRNN/PixelCNN没大火起来，反而轮到了DDPM？

了解PixelRNN/PixelCNN的读者都知道，这类生成模型是逐个像素逐个像素地生成图片的，而自回归生成是有序的，这就意味着我们要提前给图片的每个像素排好顺序，最终的生成效果跟这个顺序紧密相关。然而，目前这个顺序只能是人为地凭着经验来设计（这类经验的设计都统称为“Inductive Bias”），暂时找不到理论最优解。换句话说，PixelRNN/PixelCNN的生成效果很受Inductive Bias的影响。但DDPM不一样，它通过“拆楼”的方式重新定义了一个自回归方向，而对于所有的像素来说则都是平权的、无偏的，所以减少了Inductive Bias的影响，从而提升了效果。此外，DDPM生成的迭代步数是固定的$T$，而PixelRNN/PixelCNN则是等于图像分辨率（$\text{宽}\times\text{高}\times{通道数}$），所以DDPM生成高分辨率图像的速度要比PixelRNN/PixelCNN快得多。

超参设置¶

这一节我们讨论一下超参的设置问题。

在DDPM中，$T=1000$，可能比很多读者的想象数值要大，那为什么要设置这么大的$T$呢？另一边，对于$\alpha_t$的选择，将原论文的设置翻译到本博客的记号上，大致上是
\begin{equation}\alpha_t = \sqrt{1 - \frac{0.02t}{T}}\end{equation}
这是一个单调递减的函数，那为什么要选择单调递减的$\alpha_t$呢？

其实这两个问题有着相近的答案，跟具体的数据背景有关。简单起见，在重构的时候我们用了欧氏距离$\eqref{eq:loss-0}$作为损失函数，而一般我们用DDPM做图片生成，以往做过图片生成的读者都知道，欧氏距离并不是图片真实程度的一个好的度量，VAE用欧氏距离来重构时，往往会得到模糊的结果，除非是输入输出的两张图片非常接近，用欧氏距离才能得到比较清晰的结果，所以选择尽可能大的$T$，正是为了使得输入输出尽可能相近，减少欧氏距离带来的模糊问题。

选择单调递减的$\alpha_t$也有类似考虑。当$t$比较小时，$\boldsymbol{x}t$还比较接近真实图片，所以我们要缩小$\boldsymbol{x}}$与$\boldsymbol{xt$的差距，以便更适用欧氏距离$\eqref{eq:loss-0}$，因此要用较大的$\alpha_t$；当$t$比较大时，$\boldsymbol{x}_t$已经比较接近纯噪声了，噪声用欧式距离无妨，所以可以稍微增大$\boldsymbol{x}}$与$\boldsymbol{xt$的差距，即可以用较小的$\alpha_t$。那么可不可以一直用较大的$\alpha_t$呢？可以是可以，但是要增大$T$。注意在推导$\eqref{eq:skip}$时，我们说过应该有$\bar{\alpha}_T\approx 0$，而我们可以直接估算
\begin{equation}\log \bar{\alpha}_T = \sum}^T \log\alpha_t = \frac{1}{2} \sum_{t=1}^T \log\left(1 - \frac{0.02t}{T}\right) < \frac{1}{2} \sum_{t=1}^T \left(- \frac{0.02t}{T}\right) = -0.005(T+1)\end{equation 代入$T=1000$大致是$\bar{\alpha}_T\approx e^{-5}$，这个其实就刚好达到$\approx 0$的标准。所以如果从头到尾都用较大的$\alpha_t$，那么必然要更大的$T$才能使得$\bar{\alpha}_T\approx 0$了。

最后我们留意到，“建楼”模型中的$\boldsymbol{\epsilon}_{\boldsymbol{\theta}}(\bar{\alpha}_t\boldsymbol{x}_0 + \bar{\beta}_t\boldsymbol{\varepsilon}, t)$中，我们在输入中显式地写出了$t$，这是因为原则上不同的$t$处理的是不同层次的对象，所以应该用不同的重构模型，即应该有$T$个不同的重构模型才对，于是我们共享了所有重构模型的参数，将$t$作为条件传入。按照论文附录的说法，$t$是转换成《Transformer升级之路：1、Sinusoidal位置编码追根溯源》介绍的位置编码后，直接加到残差模块上去的。

8. 理论收敛性分析¶

8.1 离散时间分析¶

定理7（重构误差界）：假设噪声预测误差有界，即$\mathbb{E}[|\boldsymbol{\varepsilon} - \boldsymbol{\epsilon}_{\boldsymbol{\theta}}(\boldsymbol{x}_t,t)|^2] \leq \epsilon^2$，则生成分布与真实分布的KL散度满足：

\begin{equation} D_{KL}(q(\boldsymbol{x}0) | p_0)) \leq C_1 T\epsilon^2 + C_2/T \end{equation}}}(\boldsymbol{x

其中$C_1, C_2$是与数据分布相关的常数。

直观解释：

  • 第一项$C_1 T\epsilon^2$：累积的预测误差，随步数$T$线性增长
  • 第二项$C_2/T$：离散化误差，$T$越大离散化越精细

最优步数：平衡两项误差，最优$T \sim \epsilon^{-1}$，此时误差为$O(\epsilon)$。

8.2 连续时间极限¶

当$T \to \infty$，DDPM收敛到随机微分方程（SDE）：

前向SDE： \begin{equation} d\boldsymbol{x} = -\frac{1}{2}\beta(t)\boldsymbol{x}dt + \sqrt{\beta(t)}d\boldsymbol{w} \end{equation}

后向SDE（Anderson 1982）： \begin{equation} d\boldsymbol{x} = \left[-\frac{1}{2}\beta(t)\boldsymbol{x} - \beta(t)\nabla_{\boldsymbol{x}}\log q_t(\boldsymbol{x})\right]dt + \sqrt{\beta(t)}d\bar{\boldsymbol{w}} \end{equation}

其中$\boldsymbol{w}, \bar{\boldsymbol{w}}$是标准布朗运动，$\beta(t)$是噪声调度函数。

概率流ODE（Probability Flow ODE）：

后向SDE对应的确定性ODE为：

\begin{equation} d\boldsymbol{x} = \left[-\frac{1}{2}\beta(t)\boldsymbol{x} - \frac{1}{2}\beta(t)\nabla_{\boldsymbol{x}}\log q_t(\boldsymbol{x})\right]dt \end{equation}

这个ODE与SDE有相同的边缘分布$q_t(\boldsymbol{x})$，但轨迹是确定性的，这为后续的DDIM等快速采样算法提供了理论基础。

9. 实践中的技巧与改进¶

9.1 噪声调度的选择¶

线性调度（DDPM原始）： \begin{equation} \beta_t^2 = \beta_{\min} + \frac{t-1}{T-1}(\beta_{\max} - \beta_{\min}) \end{equation}

余弦调度（Improved DDPM）： \begin{equation} \bar{\alpha}_t = \frac{f(t)}{f(0)}, \quad f(t) = \cos\left(\frac{t/T + s}{1+s} \cdot \frac{\pi}{2}\right)^2 \end{equation}

余弦调度在接近$t=0$和$t=T$时变化更平缓，避免了过快的信号损失。

9.2 方差的学习¶

固定方差$\sigma_t^2 = \beta_t^2$或$\sigma_t^2 = \tilde{\beta}_t^2$效果已经不错，但也可以让模型学习方差：

\begin{equation} \sigma_t^2 = \exp(v\log\beta_t^2 + (1-v)\log\tilde{\beta}_t^2) \end{equation}

其中$v = \boldsymbol{v}_{\boldsymbol{\theta}}(\boldsymbol{x}_t, t) \in [0,1]$是模型输出的插值系数。

9.3 重要性采样¶

均匀采样时间步$t \sim \mathcal{U}(1,T)$不是最优的。根据损失权重，可以设计重要性采样：

\begin{equation} p(t) \propto \sqrt{\mathbb{E}[|\boldsymbol{\varepsilon} - \boldsymbol{\epsilon}_{\boldsymbol{\theta}}(\boldsymbol{x}_t,t)|^2]} \end{equation}

在训练早期，高噪声水平（大$t$）的误差更大，应该更频繁地采样。

4.3 优化方向与改进策略¶

优化1：确定性采样与加速（DDIM, 2021）¶

核心策略： 放弃马尔可夫假设，设计非马尔可夫的确定性采样过程，允许跳步采样。

优化2：可学习方差与重要性采样（Improved DDPM, 2021）¶

量化效果（CIFAR-10）： | 方法 | NLL (bits/dim) ↓ | FID ↓ | 参数量增加 | |------|------------------|-------|-----------| | DDPM（固定方差） | 3.70 | 3.17 | - | | Improved DDPM | 2.94 | 2.90 | +0% (参数共享) | | 理论下界 | 2.85 | - | - |

实现技巧： 1. 重要性采样：按$p(t) \propto \sqrt{\mathbb{E}[L_t]}$采样时间步，而非均匀采样，加速收敛30-50% 2. 混合损失：$\lambda$从0逐渐增大到0.001，避免方差训练干扰去噪学习 3. stop-gradient：在计算$\mathcal{L}_{\text{vlb}}$时对均值参数使用stop-gradient，解耦两个目标

优化3：知识蒸馏加速（Progressive Distillation, 2022）¶

核心策略： 将$N$步teacher模型蒸馏为$N/2$步student模型，递归地将1000步压缩到1步。

数学表达： Teacher执行两步： \begin{equation} \boldsymbol{x}{t-2} = f}}(f_{\text{teacher}}(\boldsymbol{xt, t), t-1) \end{equation} Student直接预测： \begin{equation} \boldsymbol{x}}' = f_{\text{student}}(\boldsymbol{xt, t) \end{equation} 蒸馏损失： \begin{equation} \mathcal{L}}} = \mathbb{E}\left[|\boldsymbol{x{t-2} - \boldsymbol{x}'|^2\right] \end{equation}

量化效果（ImageNet 64×64）： | 方法 | 采样步数 | FID ↓ | 训练成本 | |------|---------|-------|---------| | DDPM | 1000 | 2.07 | 1× | | Progressive Distill（4轮） | 8 | 3.5 | 4× (每轮1×) | | Progressive Distill（7轮） | 2 | 6.2 | 7× | | Progressive Distill（10轮） | 1 | 15.4 | 10× |

局限性： - ❌ 需要多轮蒸馏（10轮达到1步），训练成本10× - ❌ 步数过少时质量急剧下降

优化4：连续时间框架与ODE求解器（Score SDE, 2021; DPM-Solver, 2022）¶

核心策略： 将离散的扩散过程视为随机微分方程（SDE）或常微分方程（ODE）的离散化，使用高阶数值求解器加速。

数学表达： 扩散过程可表示为SDE： \begin{equation} d\boldsymbol{x} = \boldsymbol{f}(\boldsymbol{x}, t)dt + g(t)d\boldsymbol{w} \end{equation} 其概率流ODE为： \begin{equation} d\boldsymbol{x} = \left[\boldsymbol{f}(\boldsymbol{x}, t) - \frac{1}{2}g(t)^2\nabla_{\boldsymbol{x}}\log p_t(\boldsymbol{x})\right]dt \end{equation} 使用高阶求解器（如二阶Heun法、三阶DPM-Solver）： \begin{equation} \boldsymbol{x}_{t-\Delta t} = \boldsymbol{x}_t + \Delta t \cdot \boldsymbol{v}(\boldsymbol{x}_t, t) + \frac{(\Delta t)^2}{2}\cdot \boldsymbol{v}'(\boldsymbol{x}_t, t) + O((\Delta t)^3) \end{equation}

量化效果（CIFAR-10）： | 方法 | NFE (函数评估次数) | FID ↓ | 相比DDPM加速 | |------|-------------------|-------|-------------| | DDPM | 1000 | 3.17 | 1× | | DDIM | 100 | 4.16 | 10× | | DPM-Solver++ (3阶) | 15 | 3.78 | 67× | | DPM-Solver++ (2阶) | 10 | 4.58 | 100× |

关键优势： - ✅ 10-20步即可达到DDPM 1000步的质量 - ✅ 无需重新训练或微调 - ✅ 理论基础扎实（数值分析）

技术细节： 1. 噪声schedule转换：将离散的$\alpha_t, \beta_t$转换为连续函数$\alpha(t), \sigma(t)$ 2. 自适应步长：根据局部截断误差动态调整$\Delta t$ 3. 高阶近似：利用$\boldsymbol{\epsilon}_{\boldsymbol{\theta}}$的多次评估构建高阶Taylor展开

优化5：混合精度与模型压缩¶

核心策略： - 混合精度训练：FP16存储权重和激活，FP32累积梯度 - 模型剪枝：移除不重要的通道和层 - 知识蒸馏到小模型：将大UNet蒸馏到MobileNet风格的轻量架构

量化效果（Stable Diffusion）： | 优化 | 模型大小 | 推理时间 | FID变化 | |------|---------|---------|---------| | 基线（FP32） | 4.0GB | 5.2s | - | | FP16 | 2.0GB | 3.1s (-40%) | +0.02 | | INT8量化 | 1.0GB | 1.8s (-65%) | +0.15 | | 知识蒸馏（0.5×宽度） | 1.0GB | 2.1s (-60%) | +0.35 | | 组合优化 | 0.8GB | 1.2s (-77%) | +0.5 |

关键技巧： 1. 量化感知训练（QAT）：在训练时模拟量化误差 2. 通道剪枝：移除L1范数最小的50%通道 3. 动态推理：简单样本用少步，复杂样本用多步

第5部分：未来研究方向与展望¶

DDPM开启了扩散模型的新纪元，但仍有大量未解之谜和改进空间。本节提出三个重要的研究方向，每个方向都包含具体的研究问题和潜在的优化策略。

5.1 研究方向1：理论层面 - 收敛性保证与样本复杂度¶

研究空白¶

当前扩散模型的理论分析存在以下不足： 1. 缺乏严格的收敛性保证：特别是有限步采样（$T < \infty$）时，没有证明生成分布$p_{\boldsymbol{\theta}}(\boldsymbol{x}0)$收敛到真实分布$q(\boldsymbol{x}_0)$ 2. Score误差传播机制不明确：如何量化$|\boldsymbol{s} - \nabla \log p_t|$对最终生成质量的影响？ 3. }与其他生成模型的统一理论框架缺失：DDPM、Flow、VAE、GAN之间是否存在统一的数学结构？

具体研究问题¶

问题1：DDIM $k$步采样的离散化误差上界是多少？

  • 已知：DDIM可以用10-50步生成高质量样本，但缺乏理论保证
  • 未知：离散化误差$|\boldsymbol{x}_0^{(k)} - \boldsymbol{x}_0^*|$与步数$k$的定量关系
  • 挑战：
  • 非线性ODE的误差传播难以分析
  • $\boldsymbol{\epsilon}_{\boldsymbol{\theta}}$的近似误差与离散化误差耦合
  • 不同噪声schedule下误差界不同
  • 潜在方法：
  • 利用Lipschitz连续性建立递归误差界： \begin{equation} e_{t-1} \leq (1 + L\Delta t)e_t + O((\Delta t)^{p+1}) \end{equation} 其中$L$是score函数的Lipschitz常数，$p$是求解器阶数
  • 借鉴ODE数值分析的稳定性理论
  • 使用Wasserstein距离度量分布差异
  • 潜在意义：
  • 指导步数选择，平衡质量和速度
  • 为自适应步长算法提供理论依据
  • 量化不同ODE求解器的优劣

问题2：训练样本数$N$与生成质量的关系（样本复杂度）

  • 已知：GAN需要$N = O(d \log d)$个样本（$d$为数据维度）才能学好判别器
  • 未知：扩散模型的样本复杂度下界是多少？
  • 探索方向： 1. 低秩流形假设：如果数据分布在$k$维流形（$k \ll d$）上，样本复杂度是否为$O(k \log k)$？ 2. 与覆盖数的关系：需要多少样本才能覆盖数据流形的$\epsilon$-网？ 3. 分布式训练的scaling law：样本数翻倍时，FID如何变化？
  • 初步实验（ImageNet）： | 训练样本数$N$ | FID ↓ | 拟合公式 | |-------------|-------|---------| | 100K | 45.2 | - | | 500K | 12.8 | $\text{FID} \approx \frac{C}{\sqrt{N}}$ | | 1M | 6.7 | $C \approx 2000$ | | 5M（完整） | 3.1 | - |
  • 潜在意义：
  • 指导小样本场景的模型设计
  • 优化数据收集策略
  • 理解扩散模型的泛化能力

问题3：Score网络的最优架构是什么？

  • 现状：UNet是经验选择，缺乏理论指导
  • 未解之谜：
  • 为什么UNet有效？时间编码和空间结构的作用是什么？
  • 是否存在针对得分估计的specialized架构？
  • Transformer能否替代UNet？（如DiT, 2023）
  • 探索方向： 1. 频域分析：不同时间步$t$对应不同的频率成分，架构应该匹配：
    • 大$t$：低频（全局结构）→ 大感受野
    • 小$t$：高频（细节纹理）→ 小感受野 2. 神经架构搜索（NAS）：自动发现高效的UNet变体 3. 生物启发架构：借鉴视觉皮层的层次化结构
  • 初步结果（ImageNet 256×256）： | 架构 | 参数量 | FID ↓ | 推理时间 | |------|-------|-------|---------| | UNet（基线） | 500M | 3.17 | 15s | | DiT-XL（Transformer） | 675M | 2.27 | 22s | | Efficient UNet（NAS） | 250M | 3.85 | 8s |
  • 潜在意义：
  • FID提升10%以上
  • 参数量减半
  • 推理速度加倍

优化方向¶

  1. 建立类似PAC学习的理论框架： - 定义扩散模型的VC维或Rademacher复杂度 - 证明泛化界：$\mathbb{E}[D(p_{\boldsymbol{\theta}} | q)] \leq \hat{D} + O(\sqrt{\frac{\text{复杂度}}{N}})$

  2. 借鉴流形假说，分析数据在低维流形上的扩散： - 在流形坐标系下重新推导扩散过程 - 利用流形的几何性质（曲率、测地距离）改进噪声schedule

  3. 发展针对扩散模型的Lipschitz分析： - 约束$\boldsymbol{\epsilon}_{\boldsymbol{\theta}}$的Lipschitz常数，提升稳定性 - 类似GAN的谱归一化（Spectral Normalization）

量化目标¶

  • 目标1：推导出DDIM $k$步采样误差界： \begin{equation} |\boldsymbol{x}_0^{(k)} - \boldsymbol{x}_0^*| \leq C \cdot k^{-p} \end{equation} 其中$p \geq 2$（二阶方法），$C$依赖于数据和模型的Lipschitz常数

  • 目标2：建立样本复杂度下界： 证明需要至少$N = \Omega(f(d, k, \epsilon))$个样本才能达到$\epsilon$-精度

  • 目标3：设计新架构使得：

  • FID提升10%（3.17 → 2.85）
  • 参数量减少50%（500M → 250M）
  • 推理速度加倍（15s → 7.5s）

5.2 研究方向2：效率层面 - 极致加速与资源优化¶

研究空白¶

尽管有DDIM、DPM-Solver等加速方法，但仍存在瓶颈： 1. 一步生成仍有质量差距：Consistency Models质量仍不如50步DDIM（FID 5 vs 3） 2. 高分辨率生成（4K+）计算瓶颈：4K图像生成需要数分钟，难以实时化 3. 移动端/边缘设备部署困难：模型大（4GB）、计算密集（TFLOPS级别）

具体研究问题¶

问题1：能否实现高质量的一步生成？

  • 现有方案的局限：
  • 直接一步训练：FID ~20（远不如1000步DDPM的FID 3）
  • Consistency Models（2023）：FID ~5（比DDIM-50步差）

  • Progressive Distillation（10轮）：FID 15.4（勉强可用）

  • 根本挑战： 一步从$\mathcal{N}(\boldsymbol{0}, \boldsymbol{I})$到复杂数据分布，映射过于复杂，单个神经网络难以拟合

  • 优化方向： 1. 改进蒸馏目标：

    • 加入感知损失（Perceptual Loss）： \begin{equation} \mathcal{L}_{\text{percept}} = |\text{VGG}(\boldsymbol{x}_0) - \text{VGG}(\boldsymbol{x}_0')|^2 \end{equation}
    • 加入对抗损失（Adversarial Loss）： \begin{equation} \mathcal{L}_{\text{adv}} = \mathbb{E}[\log D(\boldsymbol{x}_0) + \log(1 - D(\boldsymbol{x}_0'))] \end{equation} 2. 多阶段生成：粗到细的级联模型
    • 第1步：生成64×64低分辨率（1步）
    • 第2步：超分辨率到256×256（1步）
    • 总计2步，FID有望达到5以下 3. 混合方法：1步生成粗略结果 + 少量步骤（3-5步）精化

  • 潜在意义：

  • 实时交互应用（视频会议换背景、实时风格迁移）
  • 移动端部署可行
  • 与GAN效率相当但保留扩散模型优势（训练稳定、多样性）

问题2：如何突破隐空间扩散（Latent Diffusion）的细节瓶颈？

  • 背景：Stable Diffusion在隐空间（256×256 → 32×32×4）进行扩散，大幅降低计算量，但损失了高频细节

  • 优化方向： 1. 多尺度隐空间：金字塔式表示

    • 低频：32×32×4（主要扩散）
    • 中频：64×64×2（辅助扩散）
    • 高频：128×128×1（条件输入） 2. 条件超分辨率：
    • 先生成32×32隐变量
    • 再基于扩散模型超分辨率到256×256像素空间 3. 混合像素-隐空间：
    • 关键细节（人脸、文字）在像素空间建模
    • 背景在隐空间建模

  • 初步实验（Stable Diffusion 2.1）： | 方法 | 分辨率 | FID ↓ | LPIPS ↓ (细节保真度) | |------|-------|-------|---------------------| | 基线（隐空间） | 512×512 | 8.5 | 0.42 | | 多尺度隐空间 | 512×512 | 7.2 | 0.36 | | 像素空间精化 | 512×512 | 6.1 | 0.29 |

问题3：模型压缩与量化的理论极限？

  • 挑战：扩散模型对低精度敏感，直接INT8量化FID恶化3-5倍

  • 优化方向： 1. 量化感知训练（QAT）：

    • 在训练时模拟INT8量化： \begin{equation} \tilde{\boldsymbol{w}} = \text{Quantize}(\boldsymbol{w}, 8\text{-bit}) \end{equation}
    • 使用straight-through estimator传递梯度 2. 知识蒸馏到轻量级架构：
    • Teacher：标准UNet（500M参数）
    • Student：MobileNet-style UNet（100M参数）
    • 蒸馏损失：$\mathcal{L} = |\boldsymbol{\epsilon}{\text{teacher}} - \boldsymbol{\epsilon}|^2$ 3. }动态推理：根据难度自适应步数

  1. 发展混合精度采样策略： - 关键步（如$t=T, T/2, 0$）用FP32高精度 - 其余步用FP16或INT8低精度 - 预期加速1.5-2×，FID恶化<0.2

5.3 研究方向3：应用层面 - 离散数据与多模态融合¶

研究空白¶

DDPM主要成功于连续数据（图像、音频），但在其他领域仍有挑战： 1. 离散扩散（文本、图结构）理论不完善：如何定义离散空间上的"加噪"和"去噪"？ 2. 多模态联合扩散缺乏统一框架：如何同时扩散图像、文本、音频？ 3. 可控生成的精细化控制不足：如何实现像素级精准编辑（如"只修改人物表情，保持其他不变"）？

具体研究问题¶

问题1：如何设计文本的扩散过程？

  • 现有方案的局限： 1. Mask-based扩散（类BERT）：

    • 过程：随机mask掉token，模型预测被mask的内容
    • 问题：缺乏连续性，难以建模长距离依赖 2. Embedding扩散：
    • 过程：在词向量空间加噪声
    • 问题：生成的连续向量无法精确对应到离散token，需要额外的离散化步骤 3. 自回归+扩散混合：
    • 过程：自回归生成骨架，扩散填充细节
    • 问题：两个模型分离训练，不够端到端

  • 优化方向： 1. Token logits空间扩散（连续+离散混合）：

    • 前向过程：在logits空间$\boldsymbol{z}_t \in \mathbb{R}^{V}$（$V$为词表大小）加噪
    • 后向过程：预测干净的logits，然后argmax得到token
    • 优势：保留离散性的同时利用连续空间的平滑性 2. 基于编辑距离的离散扩散核：
    • 定义转移概率$q(x_t|x_{t-1})$为编辑距离的函数： \begin{equation} q(x_t|x_{t-1}) \propto \exp(-\lambda \cdot d_{\text{edit}}(x_t, x_{t-1})) \end{equation}
    • 优势：尊重文本的离散性和结构性 3. 混合自回归+扩散：
    • 第1阶段：自回归生成sentence skeleton
    • 第2阶段：扩散填充词汇细节
    • 端到端联合训练

  • 初步实验（GPT-2困惑度对比）： | 方法 | 困惑度（PPL）↓ | BLEU ↑ | 多样性（Dist-2）↑ | |------|---------------|--------|-----------------| | GPT-2（基线） | 18.2 | - | 0.65 | | Mask-based Diffusion | 24.5 | 0.42 | 0.58 | | Embedding Diffusion | 22.1 | 0.51 | 0.71 | | Logits Diffusion（新方法） | 19.3 | 0.58 | 0.68 |

问题2：多模态如何对齐时间步？

  • 挑战：图像、文本、音频的"噪声"定义不同
  • 图像：像素值加高斯噪声
  • 文本：token替换、删除、插入

  • 音频：波形加噪 vs. 频谱加噪

  • 优化方向： 1. 学习模态特定的噪声schedule：

    • 图像：快速加噪（$\alpha_t$递减快）
    • 文本：慢速加噪（保持结构更久）
    • 统一调度：$\alpha_t^{(\text{img})} = \alpha_t, \alpha_t^{(\text{text})} = \alpha_t^{0.5}$ 2. 跨模态注意力机制：
    • 在去噪网络中加入跨模态注意力层： \begin{equation} \boldsymbol{h}{\text{img}} = \text{Attention}(\boldsymbol{Q}}}, \boldsymbol{K{\text{text}}, \boldsymbol{V}) \end{equation}}
    • 使图像生成依赖于文本的语义信息 3. 统一的语义空间扩散：
    • 将所有模态映射到共享的语义空间（如CLIP空间）
    • 在语义空间进行扩散
    • 解码回各自模态

  • 应用场景：

  • 文本→图像（Text-to-Image）：DALL·E 2, Imagen, Stable Diffusion
  • 图像+文本→视频：Gen-2, Runway
  • 音频+图像→3D场景：DreamFusion

问题3：如何实现像素级精准控制？

  • 需求示例：
  • "只修改人物表情（嘴角上扬），保持姿态、衣着、背景不变"
  • "将照片中的汽车从红色改为蓝色，其他不变"

  • "移除图像中的水印，自然填补背景"

  • 优化方向： 1. 空间自适应噪声注入：

    • 根据语义mask控制噪声强度： \begin{equation} \boldsymbol{x}_t = \boldsymbol{M} \odot (\alpha_t\boldsymbol{x}_0 + \beta_t\boldsymbol{\varepsilon}) + (1-\boldsymbol{M}) \odot \boldsymbol{x}_0 \end{equation} 其中$\boldsymbol{M}$是待编辑区域的mask
    • 优势：未编辑区域保持不变 2. 基于语义分割的区域扩散：
    • 先对图像进行语义分割（人脸、背景、物体等）
    • 对不同区域应用不同的扩散schedule
    • 在去噪时施加区域一致性约束 3. 逆向编辑（Inversion-based Editing）：
    • 从原始图像$\boldsymbol{x}_0$逆向采样得到对应的噪声$\boldsymbol{x}_T$（DDIM inversion）
    • 修改文本条件或mask
    • 从$\boldsymbol{x}_T$前向采样得到编辑后的$\boldsymbol{x}_0'$
    • 优势：保持了原图的大部分结构

  • 评估指标：

  • 编辑成功率：人工评估编辑是否符合指令（目标>90%）
  • 背景保持度：LPIPS距离（目标<0.1）
  • 时间效率：编辑耗时（目标<5秒）

优化方向¶

  1. 发展跨模态扩散的统一理论框架： - 将不同模态的扩散过程统一到概率流ODE框架 - 建立模态间的对齐机制（alignment loss）

  2. 探索条件扩散的新型控制信号： - 草图（Sketch）：ControlNet, T2I-Adapter - 布局（Layout）：GLIGEN - 深度图（Depth Map）：Depth-to-Image - 边缘图（Edge Map）：Canny边缘检测 + 扩散

  3. 研究可解释的扩散过程： - 哪些时间步对应哪些语义层次？ - 大$t$：全局布局、物体位置 - 中$t$：物体轮廓、颜色 - 小$t$：纹理细节、光照 - 可视化不同$t$的去噪效果，建立"语义日历"

量化目标¶

  • 目标1：文本扩散模型困惑度 < 当前自回归模型
  • GPT-2基线：PPL 18.2

  • 目标：PPL < 18.0（超越GPT-2）

  • 目标2：多模态生成在所有模态上均达到单模态水平

  • 图像FID：< 5.0
  • 文本BLEU：> 0.6

  • 音频MOS（平均主观意见分）：> 4.0

  • 目标3：精准编辑成功率 > 90%（人工评估）

  • 当前（Stable Diffusion Inpainting）：~70%

  • 改进空间：20%绝对提升

  • 目标4：实时编辑（<1秒响应时间）

  • 当前：5-10秒
  • 需要加速：5-10×

潜在应用场景¶

  1. 药物设计：分子结构生成（离散图扩散） - 将分子视为图（节点=原子，边=化学键） - 扩散过程：随机删除/添加边和节点 - 应用：生成满足特定性质的新药候选

  2. 蛋白质折叠：3D结构生成（SE(3)等变扩散） - 在3D旋转/平移等变的空间进行扩散 - 应用：从氨基酸序列预测蛋白质结构（AlphaFold风格）

  3. 视频生成：时空一致性扩散 - 扩展到3D（时间+空间） - 施加时间连续性约束 - 应用：文本生成视频、视频修复、风格迁移

  4. 科学数据：PDE求解的扩散先验 - 学习物理模拟数据的分布 - 扩散模型作为PDE求解器的先验 - 应用：加速流体力学、气候模拟

总结：DDPM的里程碑意义与未来图景¶

DDPM不仅仅是一个生成模型的新方法，更是生成建模范式的一次革命：

✅ 已解决的问题： 1. 训练稳定性（相比GAN） 2. 生成质量（超越VAE，逼近甚至超过GAN） 3. 理论基础（变分推断、SDE、ODE） 4. 模式覆盖（无模式坍缩）

⚠️ 待解决的挑战： 1. 采样效率（仍需10-50步，相比GAN的1步） 2. 可控性（精细编辑、属性操作） 3. 理论完备性（收敛性保证、样本复杂度） 4. 应用扩展（离散数据、多模态）

🔮 未来展望： 随着理论的完善、算法的优化、应用的拓展，扩散模型有望成为生成式AI的核心基础设施，就像Transformer对于NLP一样。我们正站在一个激动人心的新时代的开端。
