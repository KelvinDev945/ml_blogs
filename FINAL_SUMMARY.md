## 🎯 如何继续添加推导

### 工作流程

```bash
# 1. 选择一篇文章
vim blogs_raw/你选择的文章.md

# 2. 参考示例文章的结构
# 查看：blogs_raw/n个正态随机数的最大值的渐近估计.md

# 3. 添加推导内容
# - 使用合适的样式框
# - 长推导添加折叠
# - 嵌入相关图表（如适用）

# 4. 更新frontmatter
tags_reviewed: true  # 精心设计标签后
status: completed    # 推导完成后

# 5. 重新生成HTML
python3 scripts/generate_posts.py --incremental

# 6. 本地预览
cd docs && python3 -m http.server 8000
# 访问 http://localhost:8000/posts/你的文章.html

# 7. 确认无误后提交
git add .
git commit -m "添加推导：文章标题"
git push
```

### 推导模板

参考 `CONTRIBUTING.md` 中的写作模板：

```markdown
## 公式推导与注释

<div class="theorem-box">
### 主要结论
[正式的数学陈述...]
</div>

<div class="intuition-box">

</div>

<div class="derivation-box">
### 完整推导

<div class="step-by-step">
<div class="step">步骤1...</div>
<div class="step">步骤2...</div>
</div>

</div>

<details>
<summary>详细计算</summary>
<div markdown="1">
[复杂的中间推导...]
</div>
</details>

<div class="example-box">
### 数值例子
[具体计算...]
</div>

<iframe src="../assets/charts/...">
</iframe>
```

### 文章选择建议

**按难度分类**:

1. **简单** (推荐前5-10篇练习):
   - 单一公式推导
   - 清晰的数学结构
   - 例如：优化算法的更新规则

2. **中等** (第11-30篇):
   - 多步推导
   - 需要一定数学背景
   - 例如：损失函数的梯度计算

3. **复杂** (第31篇开始):
   - 深入的理论推导
   - 需要高级数学工具
   - 例如：随机矩阵理论、变分推断

**按主题分类**:
- 优化算法（SGD, Adam, Momentum等）
- 概率论（分布、期望、方差）
- 线性代数（矩阵分解、特征值）
- 深度学习（反向传播、归一化）
- 信息论（KL散度、互信息）

### 效率提升技巧

1. **建立个人模板库**
   - 保存常用的公式解释模式
   - 复用成功的推导结构

2. **批量处理相似文章**
   - 同一主题的文章一起处理
   - 共享数学符号和记号

3. **优先级排序**
   - 先完成重要/热门文章
   - 考虑用户可能的搜索词

4. **标签优先**
   - 即使未添加推导，也可先精心设计标签
   - 提升搜索和筛选体验
